- Data Structures
    自然人的数据信息分为多个块：自然生理信息、经历信息、关联人信息
    - 自然生理信息：
            是自然人天然所具备的数据信息，其中包含姓名、性别、生日、国籍、户籍、所在国家、所在城市、联系方式。
            其中自然人可以有多个联系方式，联系方式要有联系方式类型和联系方式信息（例如：联系方式类型：手机；联系方式信息：13666666666）联系方式类型应该可以自由配置
    - 经历信息
        是自然人过往经历的信息数据。自然人会有多段经历信息。每段经历信息默认必备字段包含开始日期、结束日期、经历类型、经历主体、经历角色、经历干系人、经历具体说明。
        - 经历类型：学习经历、工作经历、实习经历、服役经历、服刑经历等。经历类型需要可自由配置
        - 经历主体：根据不同经历类型存储不同的数据。经历主体可能是实习或工作所在单位，也有可能是学习经历中的学校名称。
        - 经历干系人：一段经历可以有多个干系人，干系人信息需要包含干系人类型、干系人姓名、干系人电话。干系人类型包含、亲属、同事、领导、客户等，干系人类型可以自由配置
    - 关联人信息
        - 关联人信息是和自然人有关系的其他自然人信息，其中包含关系类型、姓名、手机号、备注

- Functions
    自然人信息功能分为管理功能和使用功能
    - 管理功能：是对自然人信息的配置，自然生理信息可以自由增加字段；经历信息可以自由增加字段，并且可以根据经历类型定义每一个字段的label.
    - 使用功能：包含基础CRUD。在创建自然人时，根据选择的经历类型动态显示对应的自定义字段
